# Спринт D2: Установка и проверка работы PostgreSQL

## Цель
Установить PostgreSQL в Docker, проверить его работу, подключиться к базе данных и выполнить тестовые запросы.

## Задачи
1. Загрузить образ PostgreSQL 17 (официальный).
2. Создать том для хранения данных БД.
3. Запустить контейнер PostgreSQL с настройками из `settings.md`.
4. Проверить, что контейнер запущен и слушает порт 5432.
5. Подключиться к базе данных с использованием `psql` или клиента.
6. Создать тестовую таблицу и выполнить простые SQL-запросы.
7. Проверить сохранение данных после перезапуска контейнера.

## Входные данные
- Файл настроек: `/postgres/docs/settings.yaml`
- Глобальный промт: `/postgres/docs/promt_postgres.md`
- Промт текущего спринта: `/postgres/docs/D2_promt_postgres.md`
- Логи D2: `/postgres/docs/D2_logs.md`
- Все файлы каталога `/postgres/project`

## Выходные данные
- Скрипт: `/postgres/docs/D2_bash_script.py`
- Тесты: `/postgres/docs/D2_bash_script_test.py`
- Обновленный файл настроек: `/postgres/docs/settings.yaml`

## Критерии успеха
- Контейнер PostgreSQL запущен.
- Подключение к БД проходит без ошибок.
- Тестовые запросы выполняются.
- Данные сохраняются после перезапуска.
