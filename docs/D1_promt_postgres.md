# Спринт D1: Проверка работы Docker

## Цель
Убедиться, что Docker работает корректно в среде WSL2 (Ubuntu), и подготовить окружение для установки PostgreSQL.

## Задачи
1. Проверить версию Docker и его состояние.
2. Проверить наличие необходимых образов.
3. Создать тестовый контейнер (например, `hello-world`), убедиться в его работоспособности.
4. Проверить возможность монтирования томов (volumes) Docker.
5. Подготовить каталог для данных PostgreSQL (`$PROJECT_DIR/data`).

## Входные данные
- Файл настроек: `/postgres/docs/settings.yaml`
- Глобальный промт: `/postgres/docs/promt_postgres.md`
- Логи (если есть): `/postgres/docs/D1_logs.md`

## Выходные данные
- Скрипт: `/postgres/docs/D1_bash_script.py`
- Тесты: `/postgres/docs/D1_bash_script_test.py`
- Обновленный файл настроек: `/postgres/docs/settings.yaml`

## Критерии успеха
- Docker запущен и отвечает на команды.
- Контейнер `hello-world` успешно создан и завершен.
- Возможность создания тома проверена.
- Все команды выполняются без ошибок.
