
# 


Новый этап развития проекта - мы будем реализовывать devops процессы для проекта.
Сейчас есть только промышленный стенд и ручное развертывание с использованием скрипта.
Мы будем создавать контур разработки.
Будем использовать SOTA Infrastructure as Code подход: Terraform, Ansible, Docker, Docker Compose, GitHub Actions для полной автоматизации.

Важно:
- мы будем использовать MVP подход, поэтому фокус будет на простоте, лаконичности и скорости,
без преждевременной оптимизации и избыточной гибкости.
- не надо создавать избыточной документации — только минимально необходимое с лаконичным содержимым

Развивать проект будем по роадмапу небольшими итерациями (спринтами).
Создай:
- Директорию devops
- Директорию devops/doc
- Файл devops/doc/devops-roadmap.md c roadmap развития devops процессов

Роадмап должен содержать таблицу с перечнем спринтов и его параметрами.

Параметры спринта:
- Код спринта (префикс D), краткое описание, статус (название и иконка)
- Ссылка на post implementation summary файл (в нем будет описание результатов спринта)

Для каждого спринта нужна лаконичная секция, содержащая:
- Цели
- Описание состава работ тезисно (потому что для каждой итерации мы будем делать детальное планирование с помощью Cursor Plan Mode) Описание DoD (тоже лаконичное - что надо проверить 3—7 критериев)
- Список артефактов ( план, детали результатов и самые важные ADR или документация)

Состав спринтов:
Спринт 00: Погружение в проект
Цель: ознакомиться с текущей структурой и архитектурой и способом развертывания и зафиксировать текущее сос Задачи:
- Анализ структуры проекта
- Изучение текущего способа сборки и развертывания 
- Документирование текущего состояния инфраструктуры

Требования к документации:
Один документ infrastructure.md с двумя секциями:
* Текущее состояние: Mermaid диаграмма + тезисное описание что есть
* Целевое состояние: Mermaid диаграмма + роадмап
Артефакты: infrastructure.md (с двумя диаграммами)


Спринт D1: Basic Docker Setup
Цель: Запустить локально через docker-compose одной командой Задачи:
Создать простой Dockerfile для запуска (dev версия) 
- Создать docker-compose.yml для удобного развития 
- Проверить: docker-compose up - всё работает 
- Проверить главную страницу через Cursor browser Артефакты: Dockerfile, docker-compose.yml

Спринт 2: Сборка и публикация
Цель: Автоматическая сборка и публикация образов в дhcr.io Задачи:
- GitHub Actions workflow для сборки (.github/workflows/ci.yml)
- Trigger: push в main или develop ветку
- Публикация в ghcr.io/akozhin/llmstart
- Версионирование образов (tags: latest, develop)
МѴР подход (исключительно простой):
- Используется dev Dockerfile (без production оптимизаций) - Запускаем в dev режиме
- Минимальные проверки только пинтер для валидации
